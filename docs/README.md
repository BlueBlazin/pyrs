# Documentation Map

This directory is intentionally narrow.

The only primary sources of truth are:

- source code under `src/` and `include/`
- tests under `tests/`
- checked-in probe and benchmark artifacts under `perf/`

Docs in `docs/` should only do one of these jobs:

- describe a contract that is implemented in source and exercised by tests
- explain a probe/benchmark runner and point at its artifact
- record generated inventory/provenance data

Roadmap, backlog, milestone, and speculative status docs were removed during
the audit because they were not authoritative.

## Core Runtime and Validation

- `docs/CPYTHON_COMPAT_BENCHMARK.md`: full CPython `Lib/test` benchmark runner, artifact layout, and command patterns.
- `docs/PUBLIC_MICROBENCHMARKS.md`: website-facing microbenchmark suite, runner contract, and artifact layout.
- `docs/CPYTHON_COMPAT_PRIORITIES.md`: benchmark-derived execution order based on the checked-in compatibility artifact.
- `docs/BUILTIN_PARITY.md`: builtin parity gate contract tied to `scripts/check_builtin_parity.py` and `perf/builtin_parity_report.json`.
- `docs/DEVELOPER_TOOLING.md`: local tooling and validation commands that exist in this repo.
- `docs/VM_ARCHITECTURE_MAP.md`: source-backed placement guide for the current VM module layout.
- `docs/UNICODE_NAME_DATA.md`: provenance and regeneration rules for `src/parser/unicode_names_data.txt`.

## Language Accounting

- `docs/LANGUAGE_FEATURE_MANIFEST.md`: manifest gate purpose and local commands.
- `docs/LANGUAGE_FEATURE_MANIFEST.json`: machine-readable manifest consumed by `scripts/check_language_feature_manifest.py`.
- `docs/LANGUAGE_FEATURE_INVENTORY.md`: source-derived inventory summary and regeneration flow.
- `docs/LANGUAGE_FEATURE_INVENTORY.json`: generated full inventory.
- `docs/LANGUAGE_FEATURE_PROBE_MAP.json`: inventory-to-probe coverage mapping.

## Native Extensions and Scientific Stack

- `docs/EXTENSION_CAPI_V1.md`: current exported `PyrsApiV1` header/table slice from `include/pyrs_capi.h`.
- `docs/EXTENSION_CAPABILITY_MATRIX.md`: evidence-backed extension surface matrix tied to tests/probes.
- `docs/EXTENSION_PACKAGING_CONTRACT.md`: current native-extension packaging and loader contract as implemented.
- `docs/NUMPY_BRINGUP_GATE.md`: NumPy/scientific-stack probe definitions and latest checked-in artifact summary.
- `docs/CAPI_NOOP_INVENTORY.md`: current C-API no-op inventory checked by `scripts/check_capi_noop_inventory.py`.

## WASM Contracts

- `docs/WASM_API_CONTRACT.md`: JS-facing wasm export contract for `src/wasm/mod.rs`.
- `docs/WASM_CAPABILITY_MATRIX.md`: browser capability matrix and blocker-key contract.
- `docs/WASM_CLIENT_INTEGRATION_FLOW.md`: current browser call order and worker integration flow.
- `docs/WASM_MODULE_SUPPORT_POLICY.md`: source-backed module-to-blocker policy for wasm preflight.
- `docs/WASM_WORKER_RUNTIME_CONTRACT.md`: worker lifecycle/timeout/execute contract.

## Generated Artifacts and Policy Inputs

- `docs/COVERAGE_GATE_POLICY.json`: coverage gate policy consumed by `scripts/run_coverage_gate.sh`.
- `docs/NOOP_BUILTIN_INVENTORY.txt`: generated builtin no-op inventory checked by `tests/noop_inventory.rs`.
- `docs/CLONE_BASELINE.txt`: clone-count baseline generated by `scripts/clone_audit.sh`.
- `docs/CLONE_AUDIT.md`: current clone audit report generated by `scripts/clone_audit.sh`.
- `docs/DICT_BACKEND_CPYTHON_MAPPING.md`: source-backed mapping between CPython dict concepts and `src/runtime/dict_backend.rs`.
