<DocSection title="Placement Discipline">
	<ul class="docs-list">
		<li>Do not put new domain logic into monolithic orchestrator files when a focused module exists.</li>
		<li>Keep behavior in the owning subsystem (parser/compiler/vm/extensions) with minimal cross-leakage.</li>
		<li>Ship behavior changes with tests in the same commit.</li>
		<li>For CPython-visible behavior, treat CPython semantics as the sole target.</li>
	</ul>
	<Callout title="Canonical Map">
		For full ownership boundaries and placement rules, use
		<code> docs/VM_ARCHITECTURE_MAP.md</code> as the source of truth.
	</Callout>
</DocSection>
