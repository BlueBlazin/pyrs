<DocSection>
	Install PYRS with the path that matches your workflow. Most users should start with Cargo install
	from GitHub, then verify with `pyrs --version`.
</DocSection>

<DocSection title="Recommended: Cargo Install from GitHub">
	<p>This installs `pyrs` into your cargo bin and keeps command usage simple:</p>
	<CommandBlock commands={cargoInstallGitHubCommands} />
	<Callout title="Default Choice">
		Use this method unless you specifically need source hacking, containerized runs, or nightly archives.
	</Callout>
</DocSection>

<DocSection title="Choose by Workflow">
	<DataTable headers={["Need", "Best Install Path"]}>
		<tr>
			<td>Fast local setup on macOS/Linux</td>
			<td>Use <code>cargo install --locked --git ... --bin pyrs</code> (recommended default).</td>
		</tr>
		<tr>
			<td>Install from a cloned local repo</td>
			<td>Install directly from checkout with <code>cargo install --locked --path .</code>.</td>
		</tr>
		<tr>
			<td>Build directly from source without install</td>
			<td>Build via <code>cargo build --release</code>, then run <code>target/release/pyrs</code>.</td>
		</tr>
		<tr>
			<td>Containerized runs</td>
			<td>Run nightly container image <code>ghcr.io/blueblazin/pyrs:nightly</code> via Docker.</td>
		</tr>
		<tr>
			<td>No local Rust toolchain</td>
			<td>Download a nightly release archive for your platform.</td>
		</tr>
	</DataTable>
</DocSection>

<DocSection title="Alternative: Cargo Install from Local Repo Path">
