<DocSection>
	Install PYRS with the path that matches your workflow. The default channel is nightly. Use
	<code>--stable</code> when you want the latest published GitHub release.
</DocSection>

<DocSection title="Supported Native Targets" toc={false}>
	<Callout title="Platform Matrix">
		Native release binaries currently ship for macOS (<code>x86_64</code>, <code>aarch64</code>) and Linux
		(<code>x86_64</code>).
	</Callout>
</DocSection>

<DocSection title="Recommended on macOS: Homebrew (--HEAD)">
	<p>Install the tap's nightly channel with Homebrew:</p>
	<CommandBlock commands={homebrewHeadCommands} />
	<Callout title="Default Choice">
		On macOS, this is the smoothest default path and tracks the nightly channel without compiling PYRS or
		installing Homebrew's Rust toolchain. Homebrew fetches both the binary and the bundled stdlib from PYRS
		GitHub Releases, then keeps the stdlib inside the keg at <code>share/pyrs/stdlib/3.14.3/Lib</code>, so it
		does not reuse or overwrite a system CPython install. If you want the latest tagged release instead of
		nightly, drop <code>--HEAD</code>.
	</Callout>
</DocSection>

<DocSection title="Recommended on Linux (x86_64): GitHub Installer (nightly default)">
	<p>Install from GitHub Releases (default channel is nightly):</p>
	<CommandBlock commands={githubInstallerCommands} />
	<Callout title="Bundled Stdlib Policy">
		The installer puts <code>pyrs</code> in <code>~/.local/bin</code> and only stages the bundled CPython
		3.14.3 <code>Lib/</code> under <code>${"{XDG_DATA_HOME:-~/.local/share}"}/pyrs/stdlib/3.14.3/Lib</code> when
		no usable local CPython 3.14 stdlib is already present. If Python 3.14 is already installed, PYRS reuses it
		instead of dropping a second stdlib tree onto the machine.
	</Callout>
</DocSection>

<DocSection title="Optional: GitHub Installer (stable channel)">
	<p>Use this when you want the latest published GitHub release instead of nightly:</p>
	<CommandBlock commands={githubInstallerStableCommands} />
</DocSection>

<DocSection title="Choose by Workflow">
	<DataTable headers={["Need", "Best Install Path"]}>
		<tr>
			<td>Fast macOS setup</td>
			<td>Use <code>brew install --HEAD blueblazin/tap/pyrs</code>.</td>
		</tr>
		<tr>
			<td>Fast Linux setup</td>
			<td>Use the GitHub installer on Linux <code>x86_64</code> (<code>curl ... install.sh | bash</code>).</td>
		</tr>
		<tr>
			<td>Pinned stable channel</td>
			<td>Run installer with <code>--stable</code> or pin with <code>--tag vX.Y.Z</code>.</td>
		</tr>
		<tr>
			<td>Cargo-managed install</td>
			<td><code>cargo install --locked --git ... --bin pyrs</code> (requires local CPython 3.14 stdlib).</td>
		</tr>
		<tr>
			<td>Install from a cloned local repo</td>
			<td>Install directly from checkout with <code>cargo install --locked --path .</code>.</td>
		</tr>
		<tr>
			<td>Build directly from source without install</td>
			<td>Build via <code>cargo build --release</code>, then run <code>target/release/pyrs</code>.</td>
		</tr>
		<tr>
			<td>Containerized runs</td>
			<td>Run image <code>ghcr.io/blueblazin/pyrs:nightly</code> via Docker.</td>
		</tr>
	</DataTable>
</DocSection>
