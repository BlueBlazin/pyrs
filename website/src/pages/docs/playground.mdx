<DocSection>
	This page documents the experimental browser route at <a href="/playground/"><code>/playground</code></a>.
	It is a wasm probe surface for `codex/wasm`, not the production runtime contract.
</DocSection>

<DocSection title="Core Workflow">
	<DataTable headers={["Step", "Action", "What to check"]}>
		<tr>
			<td>1</td>
			<td>Click <code>Load Runtime</code></td>
			<td>Runtime status changes to loaded; runtime snapshot becomes populated.</td>
		</tr>
		<tr>
			<td>2</td>
			<td>Click <code>Inspect Runtime</code></td>
			<td>Verify backend/state/capability keys before running snippets.</td>
		</tr>
		<tr>
			<td>3</td>
			<td>Click <code>Analyze Snippet</code></td>
			<td>Review import roots and blocker rows in the preflight panel.</td>
		</tr>
		<tr>
			<td>4</td>
			<td>Click <code>Run Snippet</code></td>
			<td>Execution output includes both preflight snapshot and execute result payload.</td>
		</tr>
	</DataTable>
</DocSection>

<DocSection title="Runtime Snapshot Fields">
	<DataTable headers={["Area", "Fields", "Notes"]}>
		<tr>
			<td>Runtime</td>
			<td>
				<code>api_version</code>, <code>supports_execution</code>,
				<code>execution_backend</code>, <code>execution_blocker_count</code>
			</td>
			<td>Use these for top-level execution readiness and backend-mode branching.</td>
		</tr>
		<tr>
			<td>Worker</td>
			<td>
				<code>state</code>, <code>lifecycle_supported</code>,
				<code>execute_supported</code>, <code>timeout_configuration_supported</code>
			</td>
			<td>Use these for worker controls and state-gate interpretation.</td>
		</tr>
		<tr>
			<td>Worker Session</td>
			<td>
				<code>worker_session_info</code>, <code>worker_session_snapshot</code>
			</td>
			<td>Tracks per-session telemetry counters and last operation metadata.</td>
		</tr>
		<tr>
			<td>Capabilities</td>
			<td><code>key</code>, <code>supported</code>, <code>error</code></td>
			<td>Unsupported capabilities are explicit and map directly to blocker keys.</td>
		</tr>
	</DataTable>
</DocSection>

<DocSection title="Snippet Preflight Interpretation">
	<CodeBlock
		label="Preflight Example"
		code={`{
  "phase": "blocked_capability",
  "import_roots": ["socket"],
  "blockers": [
    {
      "module": "socket",
      "blocker_key": "network_sockets",
      "message": "network sockets are not available in browser wasm mode"
    }
  ]
}`}
	/>
	<Callout title="Execution Skip Rule" tone="warning">
		If preflight phase is not <code>supported</code>, the playground intentionally skips execution
		and reports blocker details. This prevents false-positive runtime behavior.
	</Callout>
</DocSection>
